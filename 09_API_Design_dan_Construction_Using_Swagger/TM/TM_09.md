## Tugas
Mari kita main tebak-tebakan angka acak!

Tugasmu adalah membuat API yang terdiri dari satu endpoint saja, yaitu POST /. Ketika kita melakkukan POST, formatnya adalah seperti yang sudah diberikan.

Beberapa aturan:

1. Angka acak yang dihasilkan harus tetap dan tidak boleh berubah setiap kali permintaan API dilakukan, tetapi boleh berubah setiap harinya atau dibuat tetap selamanya
2. Rentang harus di antara 1-100
3. Nama harus sensitif terhadap besar kecil huruf (mis. hamid dan Hamid akan menghasilkan angka acak yang berbeda)
4. Tidak menggunakan pustaka apapun, murni mengandalkan nama dan tebakan

Penjelasan untuk nomor 1: Jika namanya Hamid, ia akan diharapkan tetap pada nilai tebakan 24 mau kamu melakukan 100 kali permintaan. Tidak ada jawaban benar di sini (Hamid tidak harus 24, bebas mau dibuat acak seperti apa yang penting harus tetap).

## Deskripsi Program
Program ini adalah server Express yang dipakai buat game tebak angka sederhana. Saat ada request POST ke endpoint /tebak-angka, server bakal menerima dua data dari JSON, yaitu nama dan angka tebakan.

Uniknya, angka yang harus ditebak nggak dibuat secara random. Server menghitungnya dari nama yang dikirim dengan cara menjumlahkan kode ASCII dari setiap huruf. Setelah semuanya dijumlah, hasilnya dimodulo 100 lalu ditambah 1 supaya angkanya selalu berada di antara 1 sampai 100.

Angka hasil perhitungan itu kemudian dianggap sebagai jawaban yang benar. Selanjutnya server membandingkan angka tersebut dengan tebakan pengguna. Kalau tebakannya lebih kecil, muncul pesan "Tebakan terlalu rendah!". Kalau lebih besar, muncul "Tebakan terlalu tinggi!". Kalau sama persis, server mengembalikan pesan "Benar sekali yey!" sekaligus menampilkan angka yang benar.

Misalnya pada contoh, nama "Sukma" menghasilkan angka 81. Karena pengguna menebak 81, server langsung memberikan respons bahwa jawabannya benar dengan pesan "Benar sekali yey! Angka yang benar adalah 81".
