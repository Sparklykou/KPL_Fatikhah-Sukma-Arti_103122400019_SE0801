## Tugas
Jelaskan dengan kemampuanmu apa itu event delegation dalam design pattern JavaScript. Tidak ada batas bobot kata dalam menjawab tugas ini, tetapi penilaian akan bergantung dari sepaham apa dan sebagus apa kamu menyajikan jawabanmu.


## Deskripsi Program
Konsepnya, aku bikin aplikasi kantin digital yang menampilkan banyak menu makanan dari database. Setiap menu punya tombol "Tambah ke Keranjang".

Kalau tidak menggunakan Event Delegation, JavaScript harus mencari semua tombol lalu memasang addEventListener ke masing-masing tombol. Jadi kalau ada 100 menu, akan ada 100 event listener yang tersimpan di memori.

Kalau menggunakan Event Delegation, cukup pasang satu addEventListener pada elemen induk (parent) yang membungkus semua menu. Saat tombol diklik, event akan diteruskan ke parent dan JavaScript bisa mengetahui tombol mana yang ditekan. Cara ini lebih hemat memori dan lebih efisien, terutama jika jumlah menu sangat banyak atau sering bertambah secara dinamis.

### Contoh penggunaan: ini aku ambil dari salah satu potongan kode di Tubes aku 
index.html (Struktur Elemen Induk dan Anak)
HTML
```
<div id="kontainer-menu"> 
  <div class="kartu-menu">
    <span>Nasi Goreng Satria</span>
    <button class="btn-tambah" data-id="1">Tambah</button>
  </div>
  
  <div class="kartu-menu">
    <span>Mie Ayam Warmindo</span>
    <button class="btn-tambah" data-id="2">Tambah</button>
  </div>

</div>
```

app.js (Logika Event Delegation)
JavaScript
```
// 1. Ambil dulu satu elemen induknya aja
const kontainerMenu = document.getElementById('kontainer-menu');

// 2. Lalu pasang listener klik di si induk
kontainerMenu.addEventListener('click', function(event) {
    
    // 3. kita harus filter Karena kalau diklik apa aja di dalam kontainer bakal kedengaran oleh induk,
    // kita pastiin kalau dia cuma ngeksekusi kodenya kalau yang diklik tombol `btn-tambah`
    if (event.target.classList.contains('btn-tambah')) {
        
        // 4. Terus kita ngambil data ID menu dari atribut HTML 
        // (event.target ngerujuk ke tombol asli yang diklik)
        const menuId = event.target.dataset.id;
        
        5 selanjutnya tinggal meneruskan data ke rute backend
        console.log(`Mantap! Menu dengan ID ${menuId} berhasil masuk keranjang.`);
    }
});
```

Penggunaan event delegation tu ngasih keuntungan besar dalam optimalisasi performa aplikasi kantin digital aku, terutama dalam hal efisiensi memori (performance boost). Kalau pakek teknik ini, browser nggak perlu lagi buang-buang daya dan RAM buat mengurus ratusan fungsi listener di setiap tombol menu secara terpisah, melainkan cukup mengandalkan satu fungsi tunggal di elemen induknya saja agar aplikasi tetap enteng dan tidak lemot.

Selain itu, teknik ini bikin sistem jadi kebal ke perubahan konten yang bersifat dinamis. Ketika kita nambah menu makanan baru dari database menggunakan fetch API, tombol pada menu baru tersebut bakal otomatis langsung bisa diklik tanpa perlu repot melakukan bindingin ulang fungsi listener, karena si elemen induk yang posisinya tidak pernah berubah sejak awal akan selalu siap menjaga gerbang dan menangkap getaran kliknya.

Terakhir, dari sisi arsitektur kode, event delegation bikin kodingan kita jauh lebih bersih dan terstruktur. Kita tidak memerlukan lagi sintaks looping seperti `forEach` hanya untuk memasang fungsi klik satu per satu, sehingga seluruh logika handling bisa terpusat di satu tempat yang imbasnya bikin proses debugging atau pengembangan fitur ke depannya jadi jauh lebih gampang.
